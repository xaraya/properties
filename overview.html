TimeFrame Property

Description

The TimeFrame dataproperty is a widget that displays timeframes. A timeframe is defined by a start and end date. The Timeframe property displays timeframes in 2 ways:

  • A pair of jscalendar properties for start and end date respectively
  • A dropdown with predefined timeframes (e.g. 'last month')

The property's valueis an array of 3 integers. The first two are timestamps representing start and end dates. The final value is the value of the period dropdown.

Usage

Standalone

    <xar:data-input type="timeframe" [attr1="value1",...] />
    <xar:data-output type="timeframe" [attr1="value1",...] />

Bound

    <xar:data-input property="$properties['timeframe']" [attr1="value1",...] />
    <xar:data-output property="$properties['timeframe']" [attr1="value1",...] />

Examples

    <xar:data-input type="timeframe" frames="future, past" />

Attributes

  • frames: mixed [default='past'] A list or array of sets of timeframes for the periods dropdown. The sets are displayed in the dropdown in the order they are listed in the attribute. The sets 'past' and 'future' are included in the defualt templates. Unrecognized frames are ignored.
    • 'past' has the following values:
      DisplayValue
      This Month-100
      Last 7 Days-200
      Last 14 Days-300
      Last 30 Days-310
      Last 2 Months-320
      Last 3 Months-330
      Last 4 Months-340
      Last 6 Months-350
      Last 12 Months-360
      This Quarter-400
      Last Month-500
      Last 2 Months-600
      Last Quarter-700
      Year to Date-800
      Last Year-900
    • 'future' has the following values:
      DisplayValue
      Next Year500
      Till Yearend400
      Next Quarter300
      Next 2 Months200
      Next Month100
  • show: mixed [default='period'] A list or array of sets of widgets to display. Possible values 'period' and 'calendar' are included in the defualt templates. Unrecognized display types are ignored.
    Choosing 'period' displays a dropdown with predefined time frames, whicle choosing 'calendar' displays 2 calendar widgets for defining start and end dates of the time frame. If 'period' is chosen it will be given precedence in calculating start and end dates.
  • Other attributes as inherited from the parent dropdown dataproperty.
